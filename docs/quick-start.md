# test1_memo クイックスタートガイド

すぐにtest1_memoを試してみたい方向けのガイドです。

## 前提条件

- VSCodeがインストールされている
- Live Server拡張機能がインストールされている

まだの場合は、[セットアップガイド](/docs/setup-guide.md)を参照してください。

---

## ステップ1: プロジェクトを開く

1. VSCodeを起動
2. 「ファイル」→「フォルダーを開く」
3. `test1_memo` フォルダを選択

---

## ステップ2: アプリを起動

1. VSCodeのエクスプローラー（左サイドバー）で `src/index.html` を開く
2. 画面右下の「Go Live」ボタンをクリック
   - または、`index.html` 上で右クリック → 「Open with Live Server」

ブラウザが自動的に開き、test1_memoが表示されます！

---

## ステップ3: アプリを使ってみる

### 新しいメモを作成

1. 「+ 新規メモ」ボタンをクリック
2. タイトル欄にメモのタイトルを入力
3. 本文エリアにメモの内容を入力
4. 「保存」ボタンをクリック（または Ctrl+S / Cmd+S）

メモ一覧に新しいメモが追加されます。

### フォルダを作成

1. 左サイドバーの「+ 新規」ボタンをクリック
2. フォルダ名を入力して OK
3. 新しいフォルダが作成されます

### メモを検索

1. ヘッダーの検索ボックスに検索キーワードを入力
2. リアルタイムで検索結果が表示されます
3. 検索ボックスを空にすると、通常のメモ一覧に戻ります

### メモを削除

1. 削除したいメモを選択
2. 「削除」ボタンをクリック
3. 確認ダイアログで OK

---

## ステップ4: データを確認してみる

### ブラウザの開発者ツールを開く

1. ブラウザで F12 キーを押す
2. 「Console」タブを選択
3. 以下のコマンドを入力して Enter:

```javascript
debugPrintData()
```

保存されているデータの構造がコンソールに表示されます。

### LocalStorageを確認

1. 開発者ツールで「Application」タブ（Chromeの場合）または「ストレージ」タブ（Firefoxの場合）を選択
2. 左サイドバーの「Local Storage」を展開
3. 現在のURLをクリック
4. `test1_memo_data` というキーがあり、その中にデータがJSON形式で保存されているのが確認できます

## よくある質問

### Q: メモが保存されない

**A**: 以下を確認してください：
- 「保存」ボタンをクリックしたか
- ブラウザのコンソール（F12）にエラーが表示されていないか
- LocalStorageが有効になっているか（プライベートブラウジングモードでは無効）

### Q: ブラウザを閉じたらメモが消えた

**A**: LocalStorageはブラウザのデータです。以下の場合、データが消える可能性があります：
- ブラウザの閲覧履歴やキャッシュをクリアした
- プライベートブラウジングモードで使用していた
- ブラウザの設定でLocalStorageがブロックされている

**対策**: 定期的に「エクスポート機能」（Phase 3で実装予定）でバックアップを取る

### Q: Live Serverが起動しない

**A**: 以下を試してください：
- VSCodeを再起動
- Live Server拡張機能が正しくインストールされているか確認
- 手動でブラウザを開き、ファイルパスを入力（例: `file:///C:/Users/YourName/test1_memo/src/index.html`）

### Q: 画面のレイアウトが崩れている

**A**: 以下を確認してください：
- `src/css/style.css` が正しく読み込まれているか
- ブラウザの開発者ツールで「Network」タブを確認し、CSSファイルが404エラーになっていないか
- ブラウザのキャッシュをクリアしてページをリロード（Ctrl+Shift+R / Cmd+Shift+R）
